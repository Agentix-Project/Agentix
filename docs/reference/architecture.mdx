Agentix is built around a small execution loop:

```text
Host Python
  RuntimeClient.remote(fn, ...)
        |
        v
Sandbox runtime server
  Socket.IO endpoint
        |
        v
Worker subprocess
  import module
  unpickle args
  call function
  pickle result
```

Everything else supports that loop.

## Systems

| System | Responsibility |
| --- | --- |
| `agentix.runtime.shared` | Wire models, msgpack codec, framing, `RemoteCallable` |
| `agentix.runtime.client` | `RuntimeClient.remote(fn, ...)` and host-side namespace handlers |
| `agentix.runtime.server` | FastAPI and Socket.IO server, worker process management, namespace forwarding |
| `agentix.deployment` | Host-side sandbox lifecycle protocol and backend plugin lookup |
| `agentix.cli` | `agentix build` |

## Remote Target

The caller passes a normal imported function:

```python
from app import run

result = await client.remote(run, input="hello")
```

The client encodes the callable as a `RemoteCallable` import path:

```python
callable_ref = f"{run.__module__}::{run.__qualname__}"
# e.g. "app::run"
```

The worker imports the module inside the sandbox, resolves the function,
unpickles `(args, kwargs)`, calls it, and pickles the return value.

Only importable top-level functions and builtins are supported. Lambdas,
partials, bound methods, and callable instances are rejected on the host.

## Call Flow

```text
1. Host imports `app.run`.
2. Host calls `client.remote(run, input="hello")`.
3. Client builds `RemoteCallable("app::run")` and pickles `((), {"input": "hello"})`.
4. Runtime server receives the `call` Socket.IO event.
5. Server forwards the request to the worker subprocess over stdio frames.
6. Worker imports `app`, resolves `run`, unpickles args, and calls it.
7. Worker pickles the return value.
8. Server emits `call:result`; client unpickles `value`.
```

## Transports

| Path | Carries | Wire |
| --- | --- | --- |
| `GET /health` | health probe | HTTP JSON |
| Socket.IO `/` | unary RPC | `call` / `call:result` / `call:error` / `cancel` |
| Socket.IO `/trace`, `/log`, plugin paths | side channels | namespace events via worker pipe bridge |
| worker stdin/stdout | runtime ↔ worker | length-prefixed msgpack frames |

Unary RPC is the only remote-call shape implemented today. Each call
ends with one `call:result` or `call:error` keyed by `call_id`.

For host↔sandbox streaming (LLM tokens, live logs, plugin events), use
`agentix.sio` namespaces — not `client.remote`.

## Bundle Layout

`agentix build [path]` installs one Python project into the runtime venv:

```bash
/nix/runtime/bin/pip install --no-cache-dir /src/project
```

The install brings in user code, direct dependencies, transitive
dependencies, and integration packages declared in `pyproject.toml`.

```text
/nix/runtime/
├── bin/
│   ├── python
│   ├── pip
│   └── agentix-server
└── lib/python3.11/site-packages/
    ├── agentix/
    ├── agentix/bash/
    ├── agentix/swebench/
    └── app.py
```

If the project includes `default.nix`, the build adds a Nix builder stage
and links binaries into `/nix/runtime/bin`.

## Worker Model

The runtime server owns one worker subprocess. The worker handles remote
function invocation and keeps the runtime server isolated from user code.

For each call, the worker:

- resolves the `RemoteCallable` import path
- unpickles `(args, kwargs)` with stdlib pickle
- invokes the callable (awaiting coroutine results)
- pickles the return value

The worker uses the same `/nix/runtime` environment as the runtime
server, so anything installed into the bundle can be imported on demand.

## Deployment Boundary

Deployments are host-side plugins that start bundle images. They return a
`runtime_url`; after that, the runtime protocol is the same regardless of
where the sandbox is running.

```text
Deployment.create(SandboxConfig(image="python:3.13-slim", bundle="my-rollout:0.1.0"))
        |
        v
Sandbox(runtime_url="http://...")
        |
        v
RuntimeClient(runtime_url)
```

## Mental Model

```text
Bundle = what code and dependencies exist in the sandbox
Remote call = which installed function to call
Worker = where user code executes
Side-channel namespace = host↔sandbox events beyond unary RPC
Deployment = where the bundle image runs
```
