This quickstart walks the shortest end-to-end Agentix flow:

1. Declare deps in a normal Python project.
2. `agentix build` packages the project + every transitive dep into a
   self-contained bundle ‚Äî no `FROM`, nothing installed at run
   time, just a uv venv plus a Nix closure under `/nix/...`.
3. Deploy a sandbox: the bundle is bind-mounted onto a task base image
   at `/nix`. Different task images, same runtime.
4. Call a remote function with `RuntimeClient.remote(fn, ‚Ä¶)`.

## Prerequisites

- Python 3.11 or newer
- Docker or Podman for the build executor and local sandbox backend
- `uv` for dependency locking (`uv.lock` is mandatory; the build
  container runs `uv sync` against it)

## One project, two images

The two-image model is the heart of Agentix:

- **`bundle`** ‚Äî the bundle from `agentix build`. Generic and
  reusable. Holds Python, agentixx, every plugin, your code, all
  transitive deps under `/nix/store/...` with libc vendored.
- **`image`** ‚Äî the task-specific base the workload runs in (a
  SWE-bench task image, a customer environment, plain Ubuntu, ‚Ä¶).

`DockerDeployment` mounts the runtime's `/nix` into the task container
at sandbox-create time. The Python interpreter, the runtime server, your
code, and the plugin binaries (bash, coreutils, ‚Ä¶) all live under
`/nix/runtime/bin/` and resolve regardless of the task image's
distribution.

## Layout

This is the real in-repo example at `examples/hello-world` ‚Äî a flat
project: one `main.py` and one `default.nix`.

```text
hello-world/
‚îú‚îÄ‚îÄ pyproject.toml
‚îú‚îÄ‚îÄ uv.lock                       # run `uv lock`
‚îú‚îÄ‚îÄ default.nix                   # adds ripgrep (rg) to the runtime PATH
‚îú‚îÄ‚îÄ main.py                       # defines hello(); also host orchestrator
‚îî‚îÄ‚îÄ README.md
```

<Tabs>
  <Tab title="pyproject.toml">
    ```toml
    [build-system]
    requires = ["hatchling"]
    build-backend = "hatchling.build"

    [project]
    name = "hello-world"
    version = "0.1.0"
    requires-python = ">=3.11"
    dependencies = [
        "agentixx>=0.2.0",
        "agentix-runtime-basic",      # bash + files namespaces
        "agentix-deployment-docker",  # docker/podman backend
    ]

    [tool.agentix]
    nix = "default.nix"               # extra system deps for the runtime

    # Hatchling cannot infer root-level main.py from the project name, so
    # include it explicitly in the wheel that agentix build installs.
    [tool.hatch.build.targets.wheel]
    only-include = ["main.py"]
    ```
  </Tab>
  <Tab title="main.py">
    ```python

    from agentix import RuntimeClient
    from agentix.deployment.base import SandboxConfig, load_deployment, session


    def hello() -> str:

            ["rg", "--version"], check=True, capture_output=True, text=True
        )
        return proc.stdout.splitlines()[0]
    ```

    `rg` is on the sandbox PATH because `default.nix` adds it. The same
    `hello()` runs once on the host and once inside the sandbox.
  </Tab>
  <Tab title="default.nix">
    ```nix
    { pkgs }:

    pkgs.symlinkJoin {
      name = "hello-world-sys";
      paths = [ pkgs.ripgrep ];
    }
    ```

    `[tool.agentix] nix = "default.nix"` points the builder at this file.
    The builder hands it the pinned Nixpkgs and merges the result into
    `/nix/runtime/bin`.
  </Tab>
  <Tab title="uv.lock">
    Run once and commit:

    ```bash
    uv lock
    ```

    `agentix build` refuses to run without it because the builder runs
    `uv sync` against the project lock.

## Install + build

```bash
cd examples/hello-world
uv sync
uv run agentix build . --output dist/hello-world.bundle.tar
BUNDLE=$(uv run agentix deploy docker dist/hello-world.bundle.tar --format json | jq -r .bundle)
```

`agentix deploy --format json` prints
`{"bundle": ..., "platform": ..., "metadata": {...}}`, so the cache path
reads cleanly with `jq -r .bundle` instead of parsing log lines.

`agentix build` does, in order:

1. Stages your project + every plugin's `default.nix` (each plugin
   ships its system deps next to its Python module).
2. Runs Nix for the interpreter/system closures and installs the Python
   dependency closure with uv inside the build container:

   ```bash
   uv venv /nix/runtime/venv
   VIRTUAL_ENV=/nix/runtime/venv uv sync --active --frozen --no-dev --no-editable
   ```

3. Writes a portable bundle tar containing `manifest.json + nix/`.

`agentix deploy docker|podman` unpacks that tar into a content-addressed
host cache. Docker-compatible sandbox runs bind-mount the cached `nix/`
tree at `/nix`. The runtime is a plain uv venv at `/nix/runtime/venv`
plus a `symlinkJoin` of the Nix closures at `/nix/runtime/{bin,lib,...}` ‚Äî
there is no `pip` in the bundle and nothing is installed at run time.

## Call it

`main.py` doubles as the host orchestrator: it loads the Docker backend,
opens a sandbox over the deployed bundle, and runs `hello()` inside it
with `client.remote(fn, ‚Ä¶)`.

```python title="main.py"
import asyncio
import subprocess

from agentix import RuntimeClient
from agentix.deployment.base import SandboxConfig, load_deployment, session


def hello() -> str:
    """Return the bundled ripgrep version from the runtime PATH."""
    proc = subprocess.run(
        ["rg", "--version"], check=True, capture_output=True, text=True
    )
    return proc.stdout.splitlines()[0]


async def main(bundle: str) -> None:
    deployment = load_deployment("docker")()
    config = SandboxConfig(
        # Task base image ‚Äî workload environment.
        image="python:3.13-slim",
        # Cache path captured from `agentix deploy ... --format json`.
        bundle=bundle,
    )
    async with session(deployment, config) as sandbox:
        print(f"sandbox up at {sandbox.runtime_url}")
        async with RuntimeClient(sandbox.runtime_url) as client:
            print(f"Host result:    {hello()}")
            print(f"Sandbox result: {await client.remote(hello)}")
```

Run it against the bundle path captured above:

```bash
uv run python main.py --bundle "$BUNDLE"
```

Expected output (the sandbox `rg` comes from `default.nix`):

```text
sandbox up at http://localhost:33803
Host result:    ripgrep 14.1.0
Sandbox result: ripgrep 14.1.0
```

<Tip>
  For the 80% case ‚Äî Docker, a local bundle, one function ‚Äî skip the
  session boilerplate with `agentix.quick_remote`:

  ```python
  from agentix import quick_remote

  print(await quick_remote(hello, bundle="dist/hello-world.bundle.tar"))
  ```

  It loads the backend, opens a session + `RuntimeClient`, runs `fn` in a
  fresh sandbox, and returns its result (`image="python:3.13-slim"`,
  `deployment="docker"` by default).
</Tip>

## How the overlay works

Under the hood, `DockerDeployment.create` does:

```bash
# Per sandbox ‚Äî task image plus the cached runtime /nix.
docker run -d --name <sid> -p 127.0.0.1:<port>:<port> \
    -e AGENTIX_BIND_PORT=<port> \
    --mount type=bind,source="$BUNDLE"/nix,target=/nix,readonly \
    --entrypoint /nix/runtime/bootstrap.sh \
    python:3.13-slim
```

The task container sees `/nix/store/...` and `/nix/runtime/bin/...` at
their canonical paths. Python's RPATH (baked at Nix build time) finds
its libs under `/nix/store/...` because that's exactly where the
mounted cache puts them.
